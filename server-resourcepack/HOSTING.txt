Server resource pack (Java clients see enchant names and messages)
===================================================================

Players need this pack (or equivalent files) so the client can resolve
enchantment.chevvyenchants.* and chevvyenchants.command.* translation keys.
The plugin JAR alone does not send language files to clients.

Build the zip (recommended)
---------------------------

From the project root (Gradle stages pack.mcmeta from server-resourcepack/
and copies lang JSON from src/main/resources/assets/chevvyenchants/lang/):

  ./gradlew chevvyenchantsResourcePack

Output:

  build/distributions/chevvyenchants-lang.zip
  build/distributions/chevvyenchants-lang.zip.sha1   (40-char hex, no newline issues)

The build task also runs this step, so ./gradlew build produces the zip too.

Netlify auto-upload (optional)
------------------------------

If chevvyenchants.autoDeployLangPack=true, running ./gradlew chevvyenchantsResourcePack
or ./gradlew build runs deployChevvyenchantsLangPack right after the zip task: it
uploads chevvyenchants-lang.zip only when its SHA-1 changed since the last successful
deploy (tracked in .chevvyenchants-lang-last-deploy.sha1 at the repo root, gitignored).

Requires: fish, netlify CLI, scripts/mc-deploy-textures.fish, and a deploy
directory from chevvyenchants.netlifyDeployDir (gradle.properties), or
CHEVVY_NETLIFY_DEPLOY_DIR, or local.properties.

Optional when running the script alone: CHEVVY_NETLIFY_PUBLIC_URL prints the
download URL after deploy.

Disable Netlify step: chevvyenchants.autoDeployLangPack=false

Override script: chevvyenchants.deployScript=...

Manual zip (optional)
---------------------

  cd server-resourcepack/chevvyenchants-lang
  zip -r ../../../build/chevvyenchants-lang-manual.zip .

The zip must have pack.mcmeta at the top level (not inside another folder).

Host the zip
------------

Upload chevvyenchants-lang.zip somewhere that returns a direct HTTPS URL
(raw file, not an HTML page). Examples: your own site, GitHub releases,
Modrinth as a generic file, object storage with a public link.

The URL usually stays the same (same file name on Netlify or your CDN).
What must change when lang JSON or pack contents change is only the
resource-pack-sha1 line: the client uses it to know the zip bytes changed.

Gradle writes both the hash and a ready-to-merge snippet:

  build/distributions/chevvyenchants-lang.zip.sha1
  build/distributions/lang-pack-server-snippet.properties

Set chevvyenchants.resourcePackUrl in gradle.properties to your HTTPS zip URL.
Then lang-pack-server-snippet.properties includes resource-pack= and
resource-pack-sha1= for paste into server.properties.

After a successful Netlify deploy, Gradle can verify the live file matches the
local build (same check as curl URL | sha1sum vs .sha1): it runs automatically
when deployChevvyenchantsLangPack finishes if resourcePackUrl is set. Run
./gradlew verifyLangPackPublished anytime to check the URL without deploying.

Paper still needs resource-pack-sha1 updated on the server when the zip changes;
nothing can push that into server.properties for you unless you add SSH or a
host API. Use scripts/apply-lang-pack-sha-to-server-properties.fish on the host.

To patch a local server.properties in one step (after building the zip):

  ./scripts/apply-lang-pack-sha-to-server-properties.fish /path/to/server.properties

Paper allows resource-pack-sha1 empty (integrity check skipped). Clients may
still cache by URL; if players keep seeing old strings, set the SHA from the
build output instead of leaving it blank.

server.properties
-----------------

  resource-pack=https://example.com/path/chevvyenchants-lang.zip
  resource-pack-sha1=<paste from lang-pack-server-snippet.properties or .sha1 file>
  require-resource-pack=true

Optional:

  resource-pack-prompt={"text":"This server needs its translation pack."}

With require-resource-pack=true, players who refuse the prompt are disconnected.
Without it, they can join but may see raw translation keys on items.

Local testing without hosting
-----------------------------

Put chevvyenchants-lang.zip in the client resource pack folder, enable it
in the resource packs screen, then connect to the server.

Bedrock (Geyser)
----------------

Java server resource packs are not applied to Bedrock clients the same way.
Custom enchant names on Bedrock may still look wrong; gameplay stays server-side.
